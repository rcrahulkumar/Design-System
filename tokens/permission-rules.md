Permission UX Rules:

Principles:
- Explain WHY access is needed
- Show user benefit clearly
- Avoid technical language

Structure:
- Title → what permission
- Description → why it matters
- CTA → allow / settings

Guidelines:
- Use friendly tone
- Show fallback if denied
- Avoid forcing permissions

Usage:
- Onboarding → ask gradually
- Contextual → ask when needed
- Settings → allow re-trigger
