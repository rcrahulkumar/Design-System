Input System Rules:

Types:
- Use text field for short input
- Use text area for long input (journal, notes)

States:
- Default → idle
- Focus → active input
- Filled → valid data entered
- Error → validation failed
- Disabled → not interactive
- Read-only → view only

Guidelines:
- Always show label above input
- Use blue/primary border on focus
- Show error in red with message
- Avoid using disabled unless necessary

Sizes:
- Large → onboarding
- Medium → forms
- Small → cards, filters
